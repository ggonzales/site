Compatibility Testing
Agile Assurance Group’s compatibility testing is a crucial component of our global software testing services, ensuring software and applications are interoperating and integrating between operating systems, databases, networking hardware and platforms, peripheral devices, security systems and browsers.
Agile’s compatibility testing ensures integration with a host of variables to ensure the widest possible use of your application or software, including:
Operating systems
(UNIX, Windows XP, Windows Vista, Windows 7 & 8, MAC OS, MAC OSX, Linux, and mobile device operating systems).

Bowser compatibility
(Firefox, Chrome, Internet Explorer, Safari, etc.).

Networking hardware
and platforms (firewalls, proxies, etc.).

Peripherals
(drives, printers, etc.).

Databases
(Oracle, Sybase, DB2, etc.).


Compatibility testing performed by Agile Assurance Group includes backward testing; ensuring new versions of your product continue the assets of previous versions while successfully installing upgrades, or creation of easy migration paths in instances where older assets are no longer applicable. In upgrade testing, user ease in adapting to changes is critical; the feel of your product should be intuitive, using the same terminology without requiring updating of databases or the expensive necessity for new components targeted consumers.

Compatibility issues resolved by Agile’s expert, cost-efficient staff include:

Hardware
– evaluating compatibility across a variety of platforms ensures the widest possible market for your application or software. An example would be an all-hardware compatible downloadable for various brands of desktops and mobile devices.

Browsers
– Does the software or application load and unload with ease? Do all functionalities perform as desired? Are all display features uniform? Is connectivity consistent across all browsers? Is the user experience the same regardless of the browser used?

Peripherals
– Is connectivity with all systems and devices maintained? Do printers, fax machines, or in the case of mobile devices, incoming calls or message alerts perform to specification?

Version Compatibility
– Do previous and subsequent versions meld together seamlessly from the user perspective?

Software
– Does software or the application interfere with existing software of the user? Do network, messaging and operating tools continue to function?

Databases
– Do applications reliant on specific databases operate properly?

Network
– does the application or software operate on variable bandwidths? Signal strength?

User
– Agile recognizes that user dynamics across the globe are the core of today’s expanding application and software marketplace. Does User language and even intuitive aspects specific to cultures relate to your product?

Mobile devices
– with the myriad of operating systems reliant on the unique specifications around the world, does your product properly download, uninstall, and display graphics in a user-friendly fashion? Does it function properly on Android, Microsoft Windows Mobile, RIM and JME?

Cloud testing
– As cloud computing gains prevalence – that is, the means of using the internet and remote servers for data access, data management and storage resources by many new applications – compatibility testing has taken on new dimensions. Has your product been cloud tested for capacity? Stress? Load? Security?
